# Harness Engineering Notes

OpenAI の Harness Engineering 記事を、このリポジトリ向けに要約したメモです。

## このリポジトリに持ち込む要点

- AGENTS.md は短い入口にする。
- 詳細知識は repo-docs/ 以下へ分割する。
- エージェントが読めない情報は、存在しないのと同じとみなす。
- 複雑な作業は実行計画としてリポジトリ内に残す。
- ガイドラインだけでなく、将来的には設定や検証にもルールを昇格させる。
- 小規模なうちは、単純な構造と退屈な技術を優先する。

## このリポジトリ向けの補足

- docs/ は GitHub Pages の公開用フォルダとして扱う。
- リポジトリ運用の文書やテンプレートは docs/ に置かない。

## いまはまだやらないこと

- 巨大な AGENTS.md を作ること
- 先回りした shared/ や packages/ の整備
- まだ要件がない段階でのフレームワーク固定

## 次に追加しやすいもの

- 最初のアプリ用 product spec
- そのアプリの execution plan
- 採用技術に応じた lint や test の基本設定
