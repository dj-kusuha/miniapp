# Core Beliefs

## 1. AGENTS.md は地図であって、全文マニュアルではない

短い入口から必要な文書へ辿れる形を維持します。

## 2. リポジトリ外の知識は、エージェントにとって存在しない

仕様、設計意図、運用ルールは、会話だけで済ませずリポジトリ内に残します。

## 3. 小さいアプリほど単純な構造が価値を持つ

将来の柔軟性より、今の可読性と修正容易性を優先します。

## 4. 退屈な技術は強い

一般的で説明しやすく、動作を追いやすい道具を優先します。

## 5. ルールは記憶ではなく、文書や設定で強制する

繰り返し守りたいことは、会話ログではなくリポジトリに残します。

## 6. 仕様なき実装は避ける

新しいアプリや機能は、最低限の仕様を先に書きます。

## 7. 共有化は最後に行う

共通化はコストです。実際の重複が見えるまで待ちます。
