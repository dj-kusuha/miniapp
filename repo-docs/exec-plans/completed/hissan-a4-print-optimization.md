# Title

筆算ドリルの A4 印刷拡大最適化

## Status

completed

## Background

筆算ドリルの問題ページを A4 に印刷すると、数字がやや小さい一方で、単純に拡大すると 1 ページに収まりにくい。

## Scope

- docs/hissan/index.html の印刷用レイアウト調整
- A4 印刷で使う余白、問題サイズ、フィット制御の見直し
- 既存の問題/解答別印刷フローを維持したままの改善

## Non-Scope

- 問題生成ロジックそのものの変更
- 新しい依存関係や PDF 生成機能の追加
- 画面向け UI の全面刷新

## Acceptance Criteria

- 2桁どうしの筆算ページが A4 に収まりつつ、従来より大きめに印刷される
- 桁数や演算子が増えた場合も、印刷前に安全側へ自動調整される
- 既存の印刷ボタンとプレビュー挙動を壊さない

## Steps

1. 現状の印刷用 CSS と既存の iPhone 向け調整の役割を整理する
2. 印刷用のサイズ変数と自動フィット処理を追加して、A4 で使える面積を増やす
3. HTML 構文と印刷関連フローを確認し、計画を更新する

## Decision Log

- 2026-04-19: ブラウザ任せの縮小ではなく、印刷前に DOM を測定して拡大寄りのレイアウトから自動フィットする方針を採る
- 2026-04-19: 依存追加や PDF 生成には進まず、既存の HTML/CSS/JavaScript の範囲で改善する

## Validation

- `python3` の `html.parser` で `docs/hissan/index.html` の HTML 構文を確認した
- `git diff --check` で差分に不要な空白や競合がないことを確認した
- ローカルで `window.print` をスタブ化した状態で印刷ボタンを押し、印刷対象の切り替えと `afterprint` 後の通常表示復帰を確認した
- 実ブラウザの印刷プレビューでの最終的な拡大率確認は未実施のため、PR 上で A4 実機確認を依頼する
