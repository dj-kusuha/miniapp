# FRONTEND.md

## 目的

小さい Web アプリを、短く作れて、あとから直しやすい形で実装するための方針です。

## 基本方針

- モバイルファーストで設計する。
- セマンティックな HTML を優先する。
- CSS は変数を使い、色や余白を一箇所で追えるようにする。
- 小規模アプリでは依存を増やしすぎない。
- 巨大な単一ファイルを避け、責務ごとに分割する。

## UI 品質の最低条件

- 360px 幅とデスクトップ幅で破綻しない。
- キーボードだけでも主要操作ができる。
- 色だけに依存しない UI を使う。
- 非同期処理がある場合、loading と error を表示する。
- 主要フローで console error を出さない。

## 実装の目安

- 単一ページで完結するなら、素の HTML/CSS/JavaScript を優先してよい。
- 状態や部品の再利用が増えるなら、Vite と TypeScript を軸にする。
- フレームワークを入れるなら、理由を repo-docs/product-specs/ か計画に残す。

## ファイル構成の考え方

- 1 つの画面責務を複数の巨大ファイルにまたがせない。
- スタイル、マークアップ、振る舞いの境界は追いやすくする。
- コンポーネント化は重複や複雑さが見えてから行う。

## 完了前の確認

- 画面幅を変えて見たか。
- フォーカス移動が破綻していないか。
- 空状態や失敗状態が見えるか。
- 文言が仕様に沿っているか。
