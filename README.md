# miniapp
小さいWebアプリをまとめておくリポジトリ

## 現在の状態

まだ実装は始まっておらず、まずはエージェントが作業しやすい土台を整えています。

公開用の docs/ は GitHub Pages 用に予約し、リポジトリ内部の知識ベースは repo-docs/ に置きます。

公開トップの一覧ページは docs/index.html に置き、公開中の miniapp への入口として使います。

## 入口

- AGENTS.md: エージェント向けの短い地図
- repo-docs/index.md: 内部ドキュメント全体の索引
- ARCHITECTURE.md: 構造方針
- FRONTEND.md: 小さい Web アプリ向け UI 方針
- PLANS.md: 実行計画の運用ルール

## 次の進め方

新しいアプリを作るときは、先に repo-docs/product-specs/ に仕様を書き、その後 repo-docs/exec-plans/ に計画を作って実装へ進みます。
